12. OUTPUT FORMAT (TWO ROOT ARRAYS — BOTH REQUIRED)

You MUST return a single JSON object with EXACTLY these top-level keys:

1) "data" — entity-expanded array (one object per physical piece; rules 1–11 above).

2) "material_summary" — fabrication BOM / shop bill-of-materials: ONE ROW per distinct

Root shape:

{
  "data": [ /* expanded entities — one JSON object per piece */ ],
  "material_summary": [
    {
      "qty": 2,
      "material": "HSS6X6X3/16",
      "length": "8'-9 1/2\"",
      "pcmk": "3C1",
      "weight": 238,
      "grade": "A500-GR.B"
    }
  ]
}

material_summary rules (CRITICAL for fabrication alignment):

✔ Roll up identical shop items: same steel section/plate callout + same cut length + same
  usage context → one row; qty = total piece count for that line.
✔ "material": use schedule-style strings as shown on the drawings (e.g. HSS8X6X3/16,
  PL3/4\"X12\", ROD3/4\", PL1/8\"X12\") — match designation style from member/plate schedules.
✔ "length": original dimension strings only (feet-inches), same style as rule 6.
✔ "pcmk": piece mark / detail mark if shown; else null.
✔ "weight": numeric **total pounds for this line item** when computable:
   • **Prefer** weights printed on member schedules, details, or tags when shown.
   • If the drawing gives **member weight per foot** or **total weight**, copy or derive explicitly.
   • When **only** designation + cut length are shown for rolled **W / M / S / HP / MC** shapes and no tabulated weight exists, you MAY compute **estimated** piece weight as **(nominal lb/ft from the designation) × (length in feet)** — AISC designates these shapes as ``Depth×WeightPerFoot`` (second number is lb/ft). State in ``calculation_steps`` that this is nominal (e.g. ``W24×62 → 62 lb/ft × 28' = 1736 lb``). Omit if designation is ambiguous (HSS, plates, etc.) unless the drawing gives area/thickness to compute weight.
✔ "grade": steel grade as shown (e.g. A500-GR.B, A36, F1554-GR.36); else null.

✔ material_summary MUST reconcile to the same project as "data" (total steel accounted for;
  sums of qty × members must be consistent with expanded entities where applicable).

🔴 PLATE / ROD strings (common failure — fix this)

✔ For each plate, "material" is EXACTLY ONE schedule callout: thickness × width only, e.g.
  PL3/4"X12", PL1/4"X2", PL1/8"X12", PL3/16"X5 3/4". Use the same punctuation as the drawing.
❌ NEVER duplicate a width or add an extra "X…" (WRONG: PL3/4"X12"X12" — that is a typo;
  RIGHT: PL3/4"X12" and put cut length / plate height ONLY in "length").
✔ Rods / anchors: use shop-style IDs from the schedule (e.g. ROD3/4", 3/4" DIA. ANCHOR BOLT) with
  rod length or "length": null if not given — do not drop these rows from material_summary.
✔ material_summary MUST list EVERY distinct plate, rod, and washer line that appears in member/plate
  schedules or details (rolled-up qty). Omitting PL1/4", PL1/8", PL3/16", ROD3/4", etc. when they
  appear on the sheets is a failure.
